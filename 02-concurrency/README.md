# 02 — Concurrency on Go

Проекты, посвящённые конкурентному программированию в Go: горутины, каналы,
пайплайны, синхронизация через `sync.WaitGroup` и `sync.Mutex`, тикеры и
обработка системных сигналов, а также потокобезопасные структуры данных
с использованием дженериков.

## Цель

Научиться писать корректный конкурентный код: запускать и синхронизировать
горутины, строить пайплайны на каналах, грамотно завершать работу по сигналу
и проектировать потокобезопасные структуры данных без гонок (проверяется
флагом `-race`).

## Состав

| Каталог  | Задача | Что реализовано |
|----------|--------|-----------------|
| `task1`  | Гонка горутин | Запуск `N` горутин, каждая «спит» случайное время до `M` мс. После завершения всех (через `WaitGroup`) результаты сортируются по времени сна (по убыванию, при равенстве — по индексу) и печатаются. |
| `task2`  | Пайплайн на каналах | Генератор чисел в диапазоне `[K..N]` (в любом направлении) → стадия возведения в квадрат → вывод. Классический пример паттерна pipeline на каналах. |
| `task3`  | Тикер с сигналами | Каждые `K` секунд печатает номер тика и накопленное время. Корректно завершается по `SIGINT`/`SIGTERM`, печатая `Termination`. |
| `task4`  | Потокобезопасный LRU-кэш | Дженериковый `Cache[T]` с вытеснением по принципу LRU на базе `container/list` и `map`, защищённый `sync.Mutex`. Методы `Set`, `Get`, `Clear`. Покрыт юнит-тестами. |

> `task4` оформлен как отдельный Go-модуль (`task4/go.mod`) — это библиотека,
> предназначенная для использования через тесты (`go test`), без функции `main`.

## Запуск

Требуется Go 1.25+.

```bash
# task1: N горутин, максимум M мс сна
go run ./task1 -N 5 -M 200
# либо позиционно: go run ./task1 5 200

# task2: пайплайн от K до N
go run ./task2 -K 1 -N 5
# либо: go run ./task2 1 5

# task3: тикер с интервалом K секунд (Ctrl+C для остановки)
go run ./task3 -K 2
```

## Тесты

```bash
# Пайплайн
go test ./task2/

# LRU-кэш (отдельный модуль), с детектором гонок
cd task4 && go test -race ./...
```
