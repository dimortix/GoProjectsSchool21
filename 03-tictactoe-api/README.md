# 03 — Tic-Tac-Toe HTTP API

HTTP-сервис игры в крестики-нолики, в котором противником выступает компьютер
с непобедимым ИИ на алгоритме **minimax**. Проект построен по принципам чистой
архитектуры (clean architecture) с разделением на слои и внедрением зависимостей
через [Uber FX](https://github.com/uber-go/fx).

## Цель

Показать проектирование backend-приложения на Go: слоистую архитектуру с явными
границами между доменом, инфраструктурой и транспортом; DI-контейнер с управлением
жизненным циклом сервера; потокобезопасное in-memory хранилище; и реализацию
оптимального игрового ИИ через minimax.

## Архитектура

```
cmd/app            — точка входа, поднятие HTTP-сервера через fx.Lifecycle
internal/
├── domain         — доменные модели (Game, Field) и интерфейс Service
├── datasource     — реализация Service (minimax, валидация),
│                    Repository и потокобезопасное Storage (sync.Map)
├── web            — HTTP-хендлеры, DTO и мапперы domain<->web
└── di             — сборка зависимостей (fx.Module)
```

Слои общаются через интерфейсы (`domain.Service`, `datasource.Repository`),
а данные конвертируются между представлениями слоёв мапперами — это изолирует
домен от деталей транспорта и хранения.

## Игровая логика

- Поле — матрица 3×3, где `1` = X, `-1` = O, `0` = пустая клетка.
- `ValidateField` проверяет корректность хода пользователя: ровно один новый ход,
  без перезаписи чужих клеток, правильная очерёдность игрока, игра ещё не окончена.
- `NextMove` выбирает оптимальный ответный ход алгоритмом minimax (с учётом
  глубины — побеждать быстрее, проигрывать дольше), поэтому ИИ **невозможно обыграть**.
- `CheckEndGame` определяет победителя или ничью.

## Запуск

Требуется Go 1.25+.

```bash
go run ./cmd/app
# сервер слушает http://localhost:8080
```

## Использование API

`POST /game/{uuid}` — отправить текущее состояние поля со своим ходом.
Сервер валидирует ход, делает ответный ход компьютера, сохраняет состояние
и возвращает обновлённое поле.

```bash
curl -X POST http://localhost:8080/game/my-game-1 \
  -H "Content-Type: application/json" \
  -d '{"UUID":"my-game-1","Field":[[1,0,0],[0,0,0],[0,0,0]]}'
```

Пример ответа (компьютер сделал свой ход — поставил `-1`):

```json
{"UUID":"my-game-1","Field":[[1,0,0],[0,-1,0],[0,0,0]]}
```

## Технологии

- Go 1.25
- [go.uber.org/fx](https://github.com/uber-go/fx) — внедрение зависимостей и lifecycle
- `net/http` — HTTP-сервер из стандартной библиотеки
- `sync.Map` — потокобезопасное хранилище состояний игр
